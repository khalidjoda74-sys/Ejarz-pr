# تقرير إصلاح الشات الذكي

تمت مراجعة جزء الشات بعد ملاحظة أن النسخة السابقة ركزت على التقارير والترابط المالي أكثر من محرك الدردشة. هذه النسخة تضيف إصلاحات فعلية في ملفات الشات نفسها.

## ما تم تعديله

### 1. توحيد إعدادات OpenAI

تم ربط `AiChatService` بملف الإعدادات:

```text
lib/ui/ai_chat/core/ai_openai_config.dart
```

بدل الاعتماد الكامل على قراءة المفتاح من Firestore فقط.

أصبح الشات يدعم:

- خادم وسيط آمن عبر `DARFO_AI_PROXY_URL`.
- مفتاح مباشر للتجربة فقط عبر `OPENAI_API_KEY`.
- الاستمرار بدعم الطريقة القديمة `_meta/openai.api_key` للتوافق مع مشروعك الحالي.
- مهلة اتصال موحدة.
- اختيار موديل حسب نوع السؤال: تقارير، عقود، صيانة، أو سؤال عام.

### 2. تحسين الاتصال والرسائل

تم تعديل الاتصال بحيث:

- البث Streaming والاتصال العادي يستخدمان نفس headers ونفس timeout.
- الرسائل تظهر للمستخدم بشكل أوضح عند غياب الإعدادات أو انتهاء مهلة الاتصال.
- لا يظهر للمستخدم خطأ تقني خام إلا عند الحاجة.

### 3. منع دوران أدوات الشات بلا نهاية

تم وضع حد أقصى لعدد خطوات الأدوات داخل الرسالة الواحدة عبر:

```text
OPENAI_MAX_TOOL_STEPS
```

الافتراضي الآن 2 بدل خطوة واحدة، مع منع الدخول في حلقة لا تنتهي إذا أعاد النموذج طلب أدوات متكرر.

### 4. تنفيذ محلي لبعض الأوامر بدون OpenAI

أضيف مسار محلي داخل شاشة الشات يسمح بتنفيذ أوامر مهمة حتى إذا لم يكن OpenAI مهيأ بعد، مثل:

- فتح شاشة العقارات.
- فتح شاشة العملاء.
- فتح شاشة العقود.
- فتح شاشة السندات.
- فتح شاشة الصيانة.
- فتح شاشة التقارير.
- عرض ملخص الرئيسية.
- عرض قائمة العقارات.
- عرض قائمة العملاء.
- عرض قائمة العقود.
- عرض السندات غير المسددة.
- عرض الإشعارات.
- عرض تقارير العقارات والعملاء والعقود والخدمات والسندات والملاك.

هذا لا يلغي OpenAI، لكنه يجعل الشات مفيدًا ولا يتعطل تمامًا عند غياب المفتاح.

### 5. تنسيق نتائج الأدوات

تمت إضافة تنسيق محلي للنتائج حتى لا تظهر للمستخدم JSON خام أو مفاتيح تقنية قدر الإمكان.

## طريقة تشغيل الشات

### الإنتاج الموصى به

استخدم خادم وسيط:

```bash
flutter run --dart-define=DARFO_AI_PROXY_URL=https://your-domain.example.com/ai/chat
```

### التجربة المحلية فقط

```bash
flutter run \
  --dart-define=ALLOW_CLIENT_OPENAI_DIRECT=true \
  --dart-define=OPENAI_API_KEY=sk-...
```

### الطريقة القديمة

لا تزال مدعومة:

```text
Firestore collection: _meta
Document: openai
Field: api_key
```

لكن الأفضل نقل المفتاح إلى خادم وسيط لاحقًا.
